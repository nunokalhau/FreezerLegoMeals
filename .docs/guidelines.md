# Project Guidelines

## Purpose

Keep this repository simple, clear, and easy to reuse for humans and local AI tools.

## Core rules

- Keep things simple and intentional.
- Use the minimum amount of wording needed to be clear.
- Prefer short, direct wording in code, docs, and prompts.
- Use the existing structure and naming patterns.
- Make content easy to find and reuse.
- Update docs when the workflow or structure changes.

## Repository standards

- Group files by purpose.
- Use clear folder names.
- Add a short overview when a section grows.
- Prefer structured inputs and outputs such as JSON for automation.

## Content standards

- Recipes should be practical, freezer-friendly, and easy to follow.
- Docs should explain what the project does, how it is organized, and what is next.
- Keep explanations short but complete.

## Writing style

- Use short sentences.
- Avoid filler.
- Be clear and specific.
- Write for both humans and future tools.

## Contribution workflow

1. Put new content in the right place.
2. Follow the existing patterns.
3. Keep the change small and focused.
4. Update related docs if needed.
5. Check that the repository still reads clearly.

## Why this matters

This rule keeps produced code and documentation easy to understand while using the minimum amount of wording needed.
