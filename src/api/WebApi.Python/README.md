# Freezer Lego Meals - Python API

FastAPI implementation of the Freezer Lego Meals API.

This implementation aims to provide feature parity with the .NET and NestJS implementations while following Python and FastAPI best practices.

---

# Requirements

The following software is required:

| Software | Version |
|----------|---------|
| Python | 3.10 or 3.11 (recommended) |
| pip | 23+ |
| FastAPI | 0.104.x |
| Uvicorn | 0.24.x |

Important compatibility note:

- The pinned dependencies in `requirements.txt` are not currently compatible with Python 3.13.
- Use Python 3.10 or 3.11 to run this API.

Supported operating systems:

- Windows 11
- Linux
- macOS

---

# Install

Clone the repository:

```bash
git clone <repository-url>
cd FreezerLegoMeals
```

Create and activate a virtual environment:

Windows (PowerShell):

```powershell
py -3.10 -m venv .venv
.\.venv\Scripts\Activate.ps1
```

Linux/macOS:

```bash
python3.10 -m venv .venv
source .venv/bin/activate
```

Install dependencies:

```bash
pip install -r src/api/WebApi.Python/requirements.txt
```

Verify the environment (optional):

```bash
python --version
pip --version
```

---

# Run

From the project root:

```bash
py -3.10 src/api/WebApi.Python/app.py
```

Or from the `src/api/WebApi.Python` folder:

```bash
python app.py
```

Alternatively, with auto-reload:

```bash
uvicorn app:app --reload --host 0.0.0.0 --port 3000
```

The API will be available at:

```
http://localhost:3000
```

Health check:

```
http://localhost:3000/health
```

# Troubleshooting

If you get `ModuleNotFoundError: No module named 'fastapi'`:

- Make sure the virtual environment is activated.
- Reinstall dependencies with:

```bash
pip install -r src/api/WebApi.Python/requirements.txt
```

If you are on Python 3.13 and dependency installation fails:

- Switch to Python 3.10 or 3.11 and recreate the virtual environment.

# Architecture

The implementation follows a layered architecture similar to the .NET and NestJS implementations.

- FastAPI path operations expose the REST API.
- Service logic is delegated to meal and shopping service modules.
- Data access is performed through service/repository layers in the shared project structure.
- Pydantic models define request and response contracts.
- OpenAPI documentation is automatically generated by FastAPI.

The goal is to keep functional parity across API implementations while remaining idiomatic for Python.
